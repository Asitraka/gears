The main function that calls individual components.

Output is a class which inherits from the Geant4 class 'G4UImessenger'. According to the Geant4 User's Guide, all input and output should be handled through Geant4's intercom category of classes, in which G4UImessenger is included. As the name implies, all real data entry and file writing is handled by Output.

G4RunManager is sort of the 'master' class of a Geant4 simulation. When G4RunManager is created, the other major G4Managers (like G4UIManager) are also created. G4RunManager must be created and initialized in the main() method of every simulation.

The manager for visualization procedures in Geant4. Processes all visualization requests from users or functions and decided what to do with them. G4VisExecutive is the default visualization manager, and is good enough for most purposes.
